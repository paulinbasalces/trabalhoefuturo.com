# Checklist de Replicação

Use este checklist sempre que criar um novo portal.

## Estratégia

- Definir tema com demanda real de busca.
- Definir público-alvo específico.
- Definir promessa prática do portal.
- Definir 8 categorias como jornadas do usuário.
- Definir 40 a 80 itens úteis.

## Conteúdo

- Cada item tem problema claro.
- Cada item tem descrição curta e útil.
- Cada item aponta para link oficial ou confiável.
- Categorias não se sobrepõem demais.
- Há páginas de sobre e privacidade.

## SEO

- Title com nome e categoria do portal.
- Meta description com benefício claro.
- Canonical com URL pública correta.
- Open Graph preenchido.
- JSON-LD WebSite ativo.
- JSON-LD ItemList gerado pelo script.
- Sitemap atualizado.
- Robots aponta para o sitemap.

## Monetização

- GTM instalado.
- Eventos dataLayer revisados.
- Blocos de anúncio rotulados como "Publicidade".
- `ads.txt` criado somente com publisher ID real.
- Google Programmable Search ID preenchido, se houver.

## Design

- Três blocos de confiança no topo.
- Busca central funcional.
- Botão de busca Google disponível.
- Categorias em filtros rápidos.
- "Como escolhemos" em cards de confiança.
- Cards claros, escaneáveis e responsivos.
- Paleta equilibrada e adequada ao tema.

## Testes

- Abrir via servidor local HTTP.
- Confirmar que `dados.json` carrega.
- Confirmar busca local.
- Confirmar filtros.
- Confirmar modal.
- Confirmar URL com `?ferramenta=ID`.
- Confirmar páginas `sobre.html` e `privacidade.html`.
- Validar JSON e XML.
- Verificar console sem erros.

## Publicação

- Subir arquivos para GitHub.
- Ativar GitHub Pages.
- Abrir URL pública em HTTPS.
- Enviar sitemap no Google Search Console.
- Testar mobile.
- Conectar GTM/GA4/Google Ads.
- Solicitar AdSense quando houver conteúdo suficiente.
